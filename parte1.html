Parte 1 — Fundamentos y Play CDN

Tailwind v4.1 genera CSS estático a partir de las clases que realmente usas. Piensa en utilidades atómicas (p-4, text-xl, bg-blue-500) como piezas de construcción. Los tokens de @theme te permiten definir paleta y escalas reutilizables.

Paso a paso — Probar Tailwind con Play CDN

  1. Crea un archivo play.html vacío.
  2. Incluye el script del Play CDN (solo para práctica, no para producción).
  3. Declara un token de color con @theme y úsalo con text-brand.
  4. Abre el archivo en el navegador y verifica que se apliquen estilos.
<!doctype html>
<html>
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <!-- Activa Tailwind en el navegador (solo dev) -->
    <script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>
    <!-- Define tokens (solo variables o @keyframes dentro de @theme) -->
    <style type="text/tailwindcss">
      @layer theme {
        @theme {
          --color-brand: #2563eb;
        }
      }
    </style>
  </head>
  <body class="bg-gray-50 text-gray-800">
    <h1 class="text-3xl font-bold underline text-brand">Hola mundo</h1>
    <p class="mt-4 text-gray-600">Probando Tailwind con Play CDN</p>
  </body>
</html>

Ejercicio guiado — Hero con tokens y variantes

Crea un hero con un color de marca desde @theme, componentes reutilizables y soporte claro/oscuro.

  1. Define el color corporativo en @theme para exponer text-brand y bg-brand.
  2. Declara .btn y .btn-brand con @apply dentro de @layer components.
  3. Maqueta titular, párrafo y dos CTAs con utilidades de tipografía y espaciado.
  4. Usa variantes dark: para comprobar contraste y accesibilidad.
